**프로젝트**: IMEN 343 Term Project — Team #8
**문서 버전**: v0.1 (초안)
**작성일**: 2026-04-20
**담당**: 최지안 (프로토타입 개발)

---

## 1. 개요

### 1.1 목적
IMEN 343 학기 프로젝트의 실험 도구로 사용될 웹 기반 프로토타입을 개발한다. 단일 웹 애플리케이션 내에 세 가지 집중 유도 조건(C1/C2/C3)을 서로 다른 라우트로 구현하여, within-subjects 실험을 진행할 수 있게 한다.

### 1.2 범위
- 피험자 18명 대상 실험실 환경에서 사용
- 각 세션 약 25~30분 (학습 20분 + 퀴즈 + 설문)
- 본 프로토타입은 실험 도구이며, 상용 배포를 목표로 하지 않음

### 1.3 범위 외 (Non-goals)
- OS 레벨 차단 기능
- 실제 CV/활동 인식 AI 모델
- 장기 사용 추적, 계정 시스템
- 웹 프로토타입 내 유혹 요소(가짜 SNS 등) 구현 — **주의 분산 유도는 실험실에서 외부 디바이스로 처리**

## 2. 핵심 사용자 시나리오

### 2.1 실험자(Experimenter) 관점
1. 실험 시작 전, 피험자 번호를 입력하면 시스템이 조건 순서와 문제 세트 배정을 자동 할당한다 (Graeco-Latin Square).
2. 각 조건 종료 시마다 해당 세션의 로그 파일이 자동으로 다운로드된다.
3. 세션 진행 상황(현재 조건, 경과 시간)을 화면에서 확인할 수 있다.

### 2.2 피험자(Participant) 관점
1. 지정된 URL에 접속, 실험자가 입력해둔 상태에서 시작 버튼 클릭.
2. 조건별로 서로 다른 인터페이스에서 인간공학 문제 세트(A/B/C 중 하나)를 10분간 풀이한다.
3. 각 조건 종료 후 설문 페이지로 자동 이동, 작성 후 다음 조건으로 진행.

## 3. 실험 설계 (구현에 직결)

### 3.1 독립변수 (Within-subjects, 3수준)
| 조건 | 라우트 | 설명 |
|---|---|---|
| C1 | `/session/c1` | 열품타 모드 (수동 정지/재개 기능) |
| C2 | `/session/c2` | Screen Monitoring (이탈 감지 시 타이머 자동 정지 및 풀스크린 경고) |
| C3 | `/session/c3` | Screen Monitoring + 감성 피드백 (좌하단 Forest 위젯 성장/시들음) |

### 3.2 피험자 배정: 3×3 Graeco-Latin Square
피험자 18명 = 6가지 조건 순서 × 3회 반복. 각 피험자는 조건 순서와 문제 세트 배정을 동시에 받는다.

| 피험자 번호 (mod 6) | 1번 세션 | 2번 세션 | 3번 세션 |
|---|---|---|---|
| 1 | C1 × 세트A | C2 × 세트B | C3 × 세트C |
| 2 | C1 × 세트B | C2 × 세트C | C3 × 세트A |
| 3 | C2 × 세트A | C3 × 세트B | C1 × 세트C |
| 4 | C2 × 세트C | C3 × 세트A | C1 × 세트B |
| 5 | C3 × 세트A | C1 × 세트B | C2 × 세트C |
| 6 | C3 × 세트B | C1 × 세트C | C2 × 세트A |

> 7~18번 피험자는 1~6번과 동일한 배정 반복.

### 3.3 문제 세트 (학습 자료 및 퀴즈)
- **PDF 동적 생성 방식**: 실험 시작 전 PDF 3개를 업로드 (세트 A, B, C에 독립적으로 매핑)
- 업로드된 PDF에서 브라우저 내 텍스트를 추출하여 **OpenAI API**를 통해 **학습 자료(Markdown)와 퀴즈(객관식 7+서술형 3)**를 자동 생성
- 생성된 데이터는 메모리 및 localStorage에 임시 저장되어 세션 진행에 사용 (정적 JSON 방식 대체)

## 4. 기술 스택 및 아키텍처

### 4.1 스택
- **Frontend**: React 18 + Vite + TypeScript
- **Routing**: React Router v6
- **State**: React Context + useReducer (세션 상태, 로그 버퍼)
- **Styling**: Tailwind CSS (권장, 빠른 프로토타이핑)
- **Deploy**: Vercel 또는 Netlify (권장: Vercel, Vite 기본 지원)
- **백엔드 없음**: 모든 데이터는 클라이언트 메모리에서 관리 후 파일 다운로드

### 4.2 라우트 구조
```
/                           → 실험자 시작 페이지 (피험자 번호 입력)
/upload                     → PDF 업로드 및 AI 학습자료/퀴즈 자동 생성
/session/intro              → 설명 및 동의 (피험자 번호, 배정 정보 표시)
/session/c1                 → 조건 1: 열품타 모드
/session/c2                 → 조건 2: Screen Monitoring
/session/c3                 → 조건 3: Screen Monitoring + Forest
/session/survey/:condition  → 조건별 사후 설문 (NASA-TLX 등)
/session/final-survey       → 최종 비교 설문
/session/done               → 완료 화면
```

### 4.3 세션 상태 관리
```typescript
type SessionState = {
  participantId: number;           // 1~12
  assignment: {
    order: ['c1'|'c2'|'c3', ...];  // 조건 순서
    sets: ['A'|'B'|'C', ...];      // 문제 세트 배정 (순서와 동일 인덱스)
  };
  currentStep: number;             // 0: intro, 1: c1, 2: survey1, ...
  sessionStartTs: number;          // 세션별 시작 타임스탬프
  eventLog: Event[];               // 현재 세션 이벤트 버퍼
  surveyResponses: Record<string, SurveyAnswer>;
};
```

## 5. 조건별 상세 명세

### 5.1 공통 요소 (C1/C2/C3 전체)
- 상단: 남은 시간 타이머 (20:00 → 0:00, 초 단위 표시)
- 중앙: 학습 자료 (Markdown 렌더링) 표시 후, 타이머 종료 혹은 '퀴즈 시작' 클릭 시 문제 화면으로 전환
- 하단: (퀴즈 시) 진행률 바 (문제 N/M)

### 5.2 C1: 열품타 모드 (기저 조건)
- 열품타 방식의 **수동 정지 버튼** 제공. 사용자가 직접 클릭해야만 타이머가 일시정지됨.
- 다른 탭/앱으로 이탈해도 타이머는 자동으로 멈추지 않음.
- 시각 디자인: 중립적인 흑백/회색톤, 장식 최소화

### 5.3 C2: Screen Monitoring
- 다른 앱이나 탭으로 이탈 시(`visibilitychange`, `blur` 감지) **타이머 자동 정지**.
- **풀스크린 Pause Overlay(일시정지 화면)** 표시 (이탈 누적 시간, 횟수 표시), 복귀 시 자동 재개.
- 시각 디자인: 테크 느낌 (블루 계열), 집중 상태 인디케이터 제공.

### 5.4 C3: Screen Monitoring + Forest 감성 피드백
- C2와 동일하게 이탈 시 자동 정지 및 Pause Overlay 표시.
- **좌하단에 소형(160x160) Forest 위젯 고정 표시** (반짝임 등 과도한 이펙트 제거, 미니멀 디자인).
- active 상태: 나무가 천천히 성장 (씨앗→새싹→...→큰나무 5단계).
- idle/이탈 상태: 나무가 시들음 (health 수치 감소).
- 시각 디자인: 따뜻한 자연 톤 (그린/브라운).

### 5.5 '비공부' 판단 로직 (C2/C3 공통)
```
blur_threshold = 즉시 (debounce 500ms)  // 창 포커스 이탈 시 자동 일시정지 처리
idle_threshold = 10초  // 키보드/마우스 모두 비활동 (C3 시들음 및 C2 경고 유발)
```

## 6. 로깅 명세

### 6.1 이벤트 타입
모든 이벤트는 `{timestamp, type, payload}` 구조로 기록.

| 이벤트 타입 | 발생 조건 | payload |
|---|---|---|
| `session_start` | 세션 시작 | `{condition, questionSet, participantId}` |
| `session_end` | 세션 종료 | `{reason: 'timeout'|'manual'}` |
| `keypress` | 키 입력 | `{key: 'redacted'}` (내용 X, 발생만 기록) |
| `mouse_move` | 마우스 이동 (throttle 500ms) | `{x, y}` |
| `mouse_click` | 클릭 | `{target: '<selector>'}` |
| `window_blur` | 창 포커스 이탈 | `{}` |
| `window_focus` | 창 포커스 복귀 | `{}` |
| `idle_start` | idle 시작 (C2/C3 판정 시) | `{}` |
| `idle_end` | idle 종료 | `{duration_ms}` |
| `feedback_triggered` | C2 경고 / C3 나무 시들음 발동 | `{type: 'warning'|'tree_wither'}` |
| `question_answered` | 문제 답안 제출 | `{questionId, answer, timeTakenMs, isCorrect}` |
| `question_shown` | 새 문제 표시 | `{questionId}` |

### 6.2 파생 DV (분석 단계용, 로그에서 계산 가능)
- **실제 집중 시간**: 전체 세션 시간 - idle 구간 총합
- **이탈 횟수**: `idle_start` 이벤트 개수 + `window_blur` 이벤트 개수
- **집중 지속 시간**: active 구간(연속 non-idle)의 평균 및 최댓값
- **과제 완료율**: `question_answered` / 총 문제 수
- **정답률**: 보조 지표 (제안서에 없지만 로그에서 추출 가능)

### 6.3 로그 파일 다운로드
- **타이밍**: 각 조건 세션 종료 시 자동 다운로드 트리거
- **파일명 규칙**: `p{participantId:02d}_{condition}_{set}_{timestamp}.json`
  - 예: `p01_c2_B_2026-05-15T14-30-22.json`
- **파일 구조**:
```json
{
  "meta": {
    "participantId": 1,
    "condition": "c2",
    "questionSet": "B",
    "sessionOrder": 2,
    "startedAt": "2026-05-15T14:20:22.000Z",
    "endedAt": "2026-05-15T14:30:22.000Z",
    "durationMs": 600000
  },
  "events": [ /* 이벤트 배열 */ ],
  "surveyResponses": { /* 해당 조건 사후 설문 */ },
  "derivedMetrics": {
    "totalFocusTimeMs": 540000,
    "disengagementCount": 8,
    "meanFocusSpanMs": 67500,
    "completionRate": 0.85
  }
}
```

### 6.4 다운로드 실패 대응 (중요)
- 자동 다운로드가 브라우저에서 차단될 경우를 대비해, **세션 종료 화면에 "로그 재다운로드" 버튼**을 항상 제공
- 세 조건이 모두 끝난 최종 완료 화면에서 **통합 zip 재다운로드** 옵션 추가 (백업)
- 이벤트 로그는 `localStorage`에도 병렬 저장 (브라우저 종료 전까지 복구 가능)

## 7. 설문 모듈

### 7.1 조건별 사후 설문 (`/session/survey/:condition`)
- **NASA-TLX** (6개 항목, 0~100 슬라이더)
- **Flow 척도** (집중 몰입도, Likert 5점 3~5문항)
- **스트레스 수준** (Likert 5점, 단문)
- **UX 만족도** (Likert 5점, 2~3문항)
- **지속 사용 의도** (Likert 5점, 1문항)
- **감성 측정**: 성취감 / 동기부여 / 즐거움 (Likert 5점 각 1문항)

### 7.2 최종 비교 설문 (`/session/final-survey`)
- C1/C2/C3 중 가장 선호하는 조건 (강제 선택)
- 이유 자유 기술 (textarea)
- 실험 관련 코멘트 (textarea, 선택)

> 설문 문항 최종안은 이호현(실험 관리) 담당 영역. **PRD에서는 구조만 정의하고, 문항 JSON을 외부에서 주입받는 형태로 구현.**

## 8. UI/UX 요구사항

### 8.1 공통
- 반응형: 실험실 환경 고정 해상도 기준이지만, 노트북 화면 크기(1366×768 이상)에서 정상 작동
- 폰트: 한글 기본 (Pretendard 권장)
- 접근성: 색상 대비 WCAG AA 이상

### 8.2 조건 간 시각적 차별화가 명확해야 함
- 각 조건은 "다른 시스템"처럼 느껴져야 함 (시각적 priming이 실험 조작의 일부)
- 공통 레이아웃(타이머 위치, 문제 영역)은 유지하되, 색상/장식/피드백 요소로 차별화

### 8.3 실험자 페이지 (`/`)
- 피험자 번호 입력 (1~12 숫자 필드)
- 자동 계산된 배정 정보 표시 (조건 순서, 문제 세트)
- "실험 시작" 버튼 → `/session/intro`로 이동
- 접근 제어 없음 (실험자가 URL로 직접 접근, 피험자에게는 URL 미공개)

## 9. 비기능 요구사항

### 9.1 성능
- 세션 중 프레임 드랍 없이 동작 (특히 C3 나무 애니메이션)
- 이벤트 로그 버퍼가 10분 세션 동안 과도하게 커지지 않도록 mouse_move는 throttle

### 9.2 신뢰성
- 실험 도중 페이지 새로고침 시 최소한의 복구 가능성 확보
  - `sessionStorage`에 현재 진행 상태 스냅샷 저장
  - 새로고침 시 "이전 세션을 이어서 진행하시겠습니까?" 확인
- 각 세션 종료 시 로그 파일 다운로드 실패 시 사용자에게 명확한 에러 표시

## 11. 미결정 사항 (팀 논의 필요)

1. **idle_threshold 값**: 10초가 기본안. 예비 테스트에서 조정 필요.
2. **C2 경고 메시지 문구 및 빈도**: 5초간 표시 + 30초 쿨다운이 기본안.
3. **C3 나무 시각화 디테일**: 나무 종류, 애니메이션 속도, 회복 속도.
4. **문제 세트 A/B/C 구체 문항**: 신지현/박선영과 협의 필요.
5. **설문 문항 최종안**: 이호현 담당.
6. **개인정보 처리**: 피험자 번호 외 식별 정보를 수집하지 않는 것으로 명시 (IRB 준수).
